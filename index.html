Artifact 1

Title: A Timeline of Artificial Intelligence

Introduction: Artificial Intelligence as a field was conceived as a philosophical idea before the 1940s, which then materialized into practical technology. This project documented the development of Artificial intelligence at each groundbreaking epoch from the 1940s to date.

Description: This artifact is flowchart that sequences the development of AI. Alan Turing, the pioneer of modern Artificial intelligence, started the most significant stage in the development process with the Turing Test in the 1950s. Subsequent developments are visualized in this flowchart to highlight the journey of AI advancement. The chart ends with the future of AI situating in General AI development—machines that can fully reason like human beings.

Objective: Primarily, this artifact enlightens stakeholders on the pedigree of Artificial Intelligence. Contrary to factions of public opinion that AI is ‘magic’, this artifact demonstrates how thorough and established Artificial Intelligence is. There are now several journal article publications that explore several aspects of Artificial Intelligence to advance the possibility of building an human-like machine. This artifact reveals this history behind AI.

Process: This artifact is a group project from the AIML-500 class. The group of four members collaborated to produce this artifact. The group brainstormed what sources we could utilize to find accurate information. We resolved to use sources like Google, Google Scholar, and ChatGPT. After identifying the relevant information we brainstormed how to convey this information as knowledge. The group decided to use Canva to design visualization for our information.

Tools/Technologies Used: Google Scholar, ChatGPT, Gemini, Google Canva

Value Proposition: This artifact is valuable to employees, and other stakeholders who want to understand the background of Artificial Intelligence.

References: Dr. Sandra Metzger, Aaryaneil, Marilyn, Quingyuan

Artifact 2

Introduction: Artificial Intelligence development continues to elevate the user experience with chatbots. Whereas, simple chatbots would have GPT technology that makes it possible to have meaningful exchanges with chatbots. Objective: This project demonstrates the capability of an AI chatbox to guide an autonomous learning process for students.

Process: I selected the AI tool that I wanted to work with—Gemini. I selected the explore gems option, new gem. In this section I chose my chatbox name, then I inserted the parameters. After that I saved these settings and my Chatbox was ready.

Tools/Technologies Used: Gemini

Value Proposition: “Clements' Learning Assistant ” is an AI chatbox that facilitates an autonomous learning process for students and stakeholders. This chatbox makes sure to provide appropriate references to every information that the chatbox provides so that the AI generates only verifiable information. Also, the AI evaluates users on topics that the users seek to gain firm understanding. Finally, this AI chatbox guides users on a reflective process on knowledge obtained. Overall, this AI chatbox is streamlined to promote autonomous learning among users.

Artifact 4

Title: Reflection on AI (Gender Prediction) Model Project

Introduction: This project explores the application of machine learning in predicting gender based on specific human bodily features. The dataset “gender_classification_v7.csv” is selected from Kaggle; I built a predictive model through preprocessing, exploratory data analysis (EDA), and model evaluation. The project does not only demonstrate the technical process of model building but also considers the ethical implications of predicting gender using physical attributes.

Description:

Objectives: Overall, the dataset contains 5,001 rows and 8 columns of features and labels.

Feature Variables

  • long_hair
  • forehead_width_cm
  • forehead_height_cm
  • nose_wide
  • nose_long
  • lips_thin
  • distance_nose_to_lip_long

The target variable (label) represents gender, categorized into Male (0) and Female (1). I chose a Random Forest Classifier to train and validate the model. The metric used to evaluate this model is accuracy score and a confusion matrix.

Objective

The ultimate goal of this project was to develop a reliable machine learning model to predict gender using human physical features from a dataset.

Specific objectives included:

  • Conducting preprocessing and exploratory data analysis (EDA).
  • Training and testing a machine learning model for classification.
  • Evaluating performance using accuracy scores.
  • Reflecting on ethical considerations such as fairness and bias.
  • Proposing improvements for future model reliability and inclusiveness.

Process

Data Preprocessing

  • Checked for missing values, none found.
  • Converted all data types into integers.
  • Encoded gender labels (Male = 0, Female = 1).
  • Split the dataset into training and test sets.

Exploratory Data Analysis (EDA)

  • Used plots and graphs to visualize feature distributions.
  • Gained insights into patterns within the dataset.

Model Building

  • Selected Random Forest Classifier as the algorithm.
  • Trained the model using training data.

Validation and Evaluation

  • Tested predictions on unseen data.
  • Measured performance with accuracy_score.

Tools / Technologies Used

  • Python
  • Pandas, NumPy → Data cleaning and preprocessing
  • Matplotlib, Seaborn → Visualization and EDA
  • Scikit-learn → Machine learning (model training, testing, evaluation)
  • Random Forest Classifier → Selected model

Ethical Considerations

While developing this model, I ensured adherence to responsible AI practices:

  • No personal or identifiable data was used.
  • No intentional manipulation of the dataset occurred.
  • Objective, transparent methods were followed throughout the analysis.
  • Acknowledged potential racial bias in the dataset, recognizing that the lack of diversity can limit fairness and inclusivity.

Reflection and Improvements

  • Duplicate data removal during preprocessing could improve reliability.
  • Incorporating a more diverse dataset (across race and gender identity) would make the model more inclusive and ethically sound.

Value Proposition

This project highlights the workflow of building a machine learning classification model from start to finish while balancing technical accuracy with ethical responsibility. It demonstrates:

  • A practical approach to predictive modeling using physical features.
  • The importance of data quality and preprocessing in building accurate models.
  • Awareness of bias and inclusivity issues in AI systems.
  • Transferable skills that can be applied to other machine learning projects and domains.
